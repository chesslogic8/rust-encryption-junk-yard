🚀 Most Reliable File Copying
with rsync

The gold standard for moving big files & folders on Linux (especially Pop!_OS + COSMIC)

Why rsync is the most reliable way to copy files

Unlike the COSMIC file manager (which can throw “Directory not empty” errors on cross-drive copies) or even basic cp, rsync is designed for rock-solid, resumable, verifiable transfers — perfect for 11 GB files, external drives, or entire folders.

✅ Preserves permissions, timestamps, and symlinks
✅ Shows real-time progress
✅ Can resume if interrupted
✅ Skips already-copied data
✅ Can verify every byte with checksums (paranoid mode)

The Most Reliable Command (Recommended)

rsync -ah --progress --partial "/path/to/source/file-or-folder" "/path/to/destination/"
FlagWhat it does
-aArchive mode (preserves everything important)
-hHuman-readable sizes (MB/GB)
--progressReal-time percentage bar
--partialKeeps partial files so you can resume

1. Copying a Single File

# Example: Copy a big 11 GB ISO to an external drive
rsync -ah --progress --partial "/home/mark/Downloads/Ubuntu.iso" "/run/media/mark/PortableDrive/"

2. Copying an Entire Directory

Important: Add a trailing / on the source folder to copy its contents, not the folder itself.

# Copy the entire folder "MyProject" and its contents
rsync -ah --progress --partial "/home/mark/MyProject/" "/run/media/mark/BackupDrive/MyProject/"

3. Paranoid Mode: Verify Every Byte with Checksum

Use this when you want to be 100% sure the copy is bit-for-bit identical (slightly slower because it reads every file twice).

# Most reliable checksum-verified copy
rsync -ah --checksum --progress --partial "/path/to/source/" "/path/to/destination/"

After the copy finishes, you can double-check manually:

# Quick checksum verification of a single file
sha256sum "/path/to/source/file" "/path/to/destination/file"

# Or for an entire folder (generates a report)
cd /path/to/source && find . -type f -exec sha256sum {} + > source-checksums.txt
cd /path/to/destination && find . -type f -exec sha256sum {} + > dest-checksums.txt
diff source-checksums.txt ../source-checksums.txt

Bonus: Make it even easier

Add this alias to your terminal so you never have to remember the flags:

# Add to ~/.bashrc or ~/.zshrc
alias copybig='rsync -ah --progress --partial'
alias copyparanoid='rsync -ah --checksum --progress --partial'

Then just type: copybig "/source/" "/dest/"

Quick Tips for Pop!_OS Users

  • Works perfectly across internal SSD → external USB/NVMe drives
  • Works on NTFS/exFAT drives (just mount them first)
  • If interrupted, just run the exact same command again — it resumes!
  • rsync is already installed on Pop!_OS — no extra steps needed

🛡️ Most Reliable Byte-by-Byte
Disk Cloning & Backup

The modern Linux equivalent of Terabyte BootIt Bare Metal — safe, verifiable, and works perfectly on Pop!_OS + COSMIC

Why byte-by-byte copying?

Just like your old Terabyte tool, this method copies every single byte of the drive — including empty space, deleted files, partition tables, and boot sectors. Perfect for full-drive backups or cloning to another internal/portable drive.

✅ True bit-for-bit identical copy
✅ Works on Windows, Linux, or mixed drives
✅ Can verify the copy is perfect
✅ Handles large drives (11 GB+ is easy)
✅ No COSMIC file manager bugs
⚠️ CRITICAL SAFETY WARNING

# WARNING: Double-check your device names with `lsblk` first!
One wrong letter and you can wipe your entire main drive. Always run lsblk -f and confirm sizes & labels before copying.

The Most Reliable Command: dd (Simple & Rock-Solid)

# WARNING: Double-check your device names with `lsblk` first!
sudo dd if=/dev/source-disk of=/dev/dest-disk bs=4M status=progress conv=fsync

Example for a real backup:

sudo dd if=/dev/nvme0n1 of=/dev/sdb bs=4M status=progress conv=fsync
PartWhat it does
if=/dev/source-diskSource drive (the one you want to copy FROM)
of=/dev/dest-diskDestination drive (the one you want to copy TO)
bs=4MFast 4-megabyte block size
status=progressLive progress bar
conv=fsyncEnsures all data is written before finishing

After the copy: Verify it's perfect

# Quick byte-by-byte comparison (fastest check)
sudo cmp -s /dev/source-disk /dev/dest-disk && echo "✅ Perfect match!" || echo "❌ Mismatch found!"

Or use checksums for extra peace of mind (slower but 100% sure):

sudo sha256sum /dev/source-disk > source.sha256
sudo sha256sum /dev/dest-disk > dest.sha256
diff source.sha256 dest.sha256 && echo "✅ Checksums match!"

Better for damaged drives: ddrescue

# Install once: sudo apt install gddrescue
sudo ddrescue -f -r3 /dev/source-disk /dev/dest-disk rescue.log

ddrescue is smarter — it retries bad sectors and creates a log so you can resume later.

Easiest Option: Rescuezilla (GUI, like Terabyte Bare Metal)

  1. Download Rescuezilla ISO from rescuezilla.com
  2. Flash it to a thumb drive with Balena Etcher or dd
  3. Boot from the thumb drive (press F12 or Esc during boot)
  4. Choose “Clone” or “Image” → select source & destination drives
  5. It does byte-by-byte copy + built-in verification

Rescuezilla has nice safety dialogs and is exactly what most people use instead of the old Terabyte tool in 2026.

Quick Tips for Pop!_OS Users

  • Always run lsblk -f first — it shows names, sizes, and filesystem labels
  • Unmount drives before cloning: sudo umount /dev/sdb1 (replace with actual partitions)
  • For BitLocker/Windows drives: suspend BitLocker in Windows first
  • External drives work great — just make sure the destination is at least as big as the source
  • These commands work in the Pop!_OS terminal — no extra installs needed for dd
